# AI PPT 版面生成提示词合集

用大模型（Claude、GPT、Gemini、Kimi 等）把一堆文字变成一页页版式好看、逻辑清晰的演示文稿，是目前最实用的 AI 应用场景之一。这份合集按照「**内容 → 大纲 → 版式 → 图表 → 讲稿**」的制作顺序整理了 6 个可以直接使用的提示词，覆盖从零散素材到成品页面的完整流程。

## 使用建议

| 场景 | 使用哪个提示词 | 备注 |
| --- | --- | --- |
| 手里只有一堆资料/会议记录，还没想清楚讲什么 | [大纲与叙事结构生成](#_2) | 先理逻辑，再做版式 |
| 只需要做一张封面页或单页信息图 | [单页 HTML 版式设计](#html) | 输出可直接用浏览器打开、打印成 PDF |
| 要做一整套风格统一的多页幻灯片 | [多页系列版式设计](#_3) | 先定设计系统，再批量生成每一页 |
| 页面里有数据、指标、对比关系 | [数据可视化图表页版式](#_4) | 内置图表库，避免手绘图表错位 |
| 已经有 PPT 草稿，需要精简内容 | [内容整理与精炼](#_5) | 适合把长文档压缩成要点 |
| 页面做完了，还差演讲串词 | [演讲稿与备注生成](#_6) | 按页生成讲者备注 |

推荐工作流：先用「大纲生成」理清结构 → 用「单页/多页版式设计」把每页做成 HTML → 需要图表的页面单独用「数据可视化」提示词精修 → 最后用「演讲稿生成」补上讲者备注。生成的 HTML 页面可以直接用浏览器打开，通过打印为 PDF 导出，或截图导入 PPT。

## 大纲与叙事结构生成

先把素材整理成有逻辑的大纲，再进入排版环节，避免出现内容堆砌、页面之间跳跃的问题。

```
你是一名资深的演示文稿策划师，擅长把零散信息整理成有逻辑、有节奏的叙事结构。

请根据我提供的内容，输出一份 PPT 大纲，要求：

1. 先用一句话提炼本次演示的核心论点（Big Idea），所有页面都要为这句话服务
2. 按照「开场引入 - 问题/背景 - 核心内容 - 论证/案例/数据 - 结论与行动点」的节奏组织结构，可根据内容适当调整
3. 拆分出具体页面，每页给出：页面标题、本页要传达的一个核心观点（一句话，不是多个要点堆砌）、支撑这个观点的关键信息（3-5 条以内）、建议的呈现方式（文字列表 / 对比 / 时间线 / 数据图表 / 图片配文字等）
4. 控制页面总数在 {页数} 页左右，每页信息量以「10 秒内能看懂」为标准做取舍
5. 用 Markdown 的多级标题输出，方便我后续逐页确认和调整

如果我提供的内容里存在逻辑跳跃、证据不足或数据来源不明确的地方，请直接指出，不要替我编造内容。

我要制作的演示主题是：{主题}
目标观众是：{观众，例如：投资人 / 团队内部 / 客户 / 课堂}
我的素材如下：
"""
{在此粘贴素材，可以是会议记录、文档、笔记等}
"""
```

## 单页 HTML 版式设计

这是目前最常用的「AI 排版」玩法：用一份带设计规范的提示词，让模型直接输出一个自包含的 HTML 页面，比让模型描述「配色建议」再自己动手排版效率高得多。

```
你是一名专精信息设计（Information Design）的资深视觉设计师，请把我提供的内容设计成一张 16:9 的演示文稿页面，用单个 HTML 文件实现。

## 画布与网格
- 画布尺寸固定为 1280×720px（16:9），页面内容禁止溢出画布或出现滚动条
- 采用网格系统排版，页面四周留出至少 48px 的安全边距
- 同一层级的元素要对齐，避免随意堆砌

## 视觉设计系统
- 整体配色控制在 1 个主色 + 1 个辅助色 + 1 个强调色 + 中性色（黑/白/灰）以内，不要出现风格不统一的多种颜色
- 字体层级最多 3 级：大标题、小标题/要点、正文说明，字号、字重要有明显区分
- 用图标（Font Awesome 或语义化的 SVG）辅助信息识别，不要用无意义的装饰性图形
- 如果内容包含数字/百分比等关键指标，用视觉上更突出的方式呈现（大字号、卡片、色块等），而不是和普通正文混排

## 技术实现
- 输出一个完整可直接用浏览器打开的 HTML 文件，通过 CDN 引入 Tailwind CSS 和 Font Awesome（或等价的图标库），不要写内联的大段自定义 CSS
- 不使用任何占位图片或 Lorem Ipsum，文字内容必须完全来自我提供的素材，不要编造数据或案例
- 页面右下角保留一个小的页码/来源角标位置

## 内容要求
- 页面类型：{封面页 / 目录页 / 内容页 / 结尾页，按需选择}
- 页面标题：{标题}
- 核心内容：
"""
{在此粘贴该页要呈现的内容，尽量精简，1 页对应 1 个核心观点}
"""

请先用一两句话说明你打算采用的版式思路（例如左右分栏、卡片网格、时间线等），再输出完整代码。
```

## 多页系列版式设计

做一整套 PPT 时，最容易出问题的是「每一页都很好看，但连在一起风格不统一」。解决办法是先定一份设计系统说明书，再让模型严格按照它逐页生成。

```
你是一名视觉设计师，我需要制作一套风格统一的多页 HTML 演示文稿，请分两步完成。

## 第一步：先输出「设计系统说明书」
在开始设计具体页面前，先根据以下主题和调性，制定一份简短的设计规范，包含：
- 主色 / 辅助色 / 强调色的具体色值（HEX）
- 标题字体、正文字体（Web 安全字体或可通过 CDN 引入的字体）
- 标题、小标题、正文、注释文字的字号与字重
- 通用的页眉/页脚样式（如页码位置、章节标识位置）
- 常用的信息呈现组件（例如：要点卡片、数据卡片、对比框、时间线样式），并说明各组件的圆角、间距、阴影等细节

演示主题：{主题}
风格调性：{例如：科技感 / 商务简约 / 温暖手绘 / 学术严谨}

## 第二步：逐页生成
在我确认设计系统后，请严格按照该规范，为下面提供的每一页内容分别生成一个 1280×720px 的 HTML 文件（16:9，禁止内容溢出画布），页面之间只允许在具体内容和局部版式（分栏 / 卡片 / 时间线等）上有差异，配色、字体、页眉页脚等设计系统元素必须保持完全一致。

每一页请按「页码 - 标题 - 内容」的格式提供给你：
"""
{按页粘贴内容，例如：
第 1 页（封面）：{标题/副标题/演讲人信息}
第 2 页（目录）：{章节列表}
第 3 页：{标题} — {要点}
...}
"""

如果某一页内容偏多，无法在一页内清晰呈现，请提示我拆分为两页，而不是硬塞进一页导致溢出。
```

## 数据可视化图表页版式

涉及数据对比、趋势、占比的页面，直接让模型手绘图表容易出现比例失真，改用图表库生成更可靠。

```
你是一名擅长数据可视化的信息设计师，请把以下数据设计成一张 16:9 的 HTML 仪表盘风格页面。

## 要求
- 画布 1280×720px，通过 CDN 引入 ECharts（或 Chart.js），图表使用真实的 <canvas>/<div> 渲染，不要用图片或手绘 SVG 模拟图表
- 根据数据类型选择合适的图表：趋势用折线图、占比用环形图/条形图、对比用柱状图、少量关键指标用数字卡片（KPI Card）而不是图表
- 图表配色需要和页面的主色/辅助色保持一致，不要使用图表库的默认配色
- 每个图表要有清晰的标题和数据来源标注（如果我提供了来源）
- 数字必须和我提供的数据完全一致，不要四舍五入、编造或美化数据
- 页面整体采用网格布局（例如 1 个主图表 + 2-3 个 KPI 卡片），避免图表和文字大段混排

数据与结论如下：
"""
{在此粘贴数据表格、关键指标、以及你希望强调的结论}
"""

请先说明你为每组数据选择的图表类型和理由，再输出完整 HTML 代码。
```

## 内容整理与精炼

拿到一份冗长的文档或已有 PPT 草稿，先做减法，再进入版式设计。

```
你是一名信息整理专家，请帮我把下面的内容精炼成适合放进 PPT 页面的要点。

要求：
1. 先判断这份内容的核心论点是什么，无关信息直接剔除，不要试图保留所有细节
2. 每个要点控制在一行以内（15-20 个字），不写完整句子，用短语呈现
3. 每页要点数量不超过 5 条，超过的部分拆分成多页或建议合并到下一页的补充说明
4. 如果原文中有具体数字、案例，优先保留这些具体信息而不是概括性描述
5. 按「页面标题 - 要点列表」的格式输出，方便直接套用到版式设计提示词里

原文内容：
"""
{在此粘贴原文}
"""
```

## 演讲稿与备注生成

页面做完之后，按页生成讲者备注，避免临场词不达意。

```
你是一名演讲教练，请根据下面每一页 PPT 的内容，为演讲者写一份逐页的讲者备注（Speaker Notes）。

要求：
1. 每页备注控制在 30-60 秒的口语讲述量（约 80-150 字），不要照读页面文字，而是补充页面上没有写出来的解释、案例或过渡语
2. 页面之间要有自然的承接句，让整场演讲听起来是连贯的叙事，而不是一页页孤立念稿
3. 在需要停顿强调、互动提问或展示演示的地方，用「【提示：...】」标出来
4. 语气：{正式汇报 / 轻松分享 / 说服性演讲，按需选择}

PPT 页面内容如下（按页提供）：
"""
{按页粘贴每页标题和要点}
"""
```
