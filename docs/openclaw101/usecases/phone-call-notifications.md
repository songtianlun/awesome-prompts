# 电话通知

你的智能体已经在监控各种东西 —— 股票、邮件、智能家居、日历 —— 但通知很容易被忽略。推送通知堆积如山。聊天消息被埋没。对于真正重要的事情，你需要一种无法一划而过的通知。

这个用例给你的智能体一个电话通话作为通知渠道。当某事足够紧急时，智能体拨打你的真实手机号，告诉你发生了什么，你可以回话。双向对话，不是机器人电话。

## 功能

- 智能体判定某事值得你关注（价格告警、紧急邮件、预约提醒等）
- 智能体通过 [clawr.ing](https://clawr.ing) 拨打你的手机 —— 无需 Twilio 设置，无需配置 API 密钥
- 你接听，听到告警，可以实时问后续问题
- 对话结束时通话结束

关键思想：智能体打给你。你不打给智能体。这适用于心跳检查、定时任务或任何触发器 —— 智能体评估某事是否值得打电话，然后执行。

## 所需技能

- [clawr.ing](https://clawhub.ai/marcospgp/clawring) —— 从 [clawr.ing 仪表板](https://clawr.ing) 粘贴设置提示词到 OpenClaw 聊天即可。无需 CLI 安装。

## 示例：早间简报电话

```text
每个工作日早上 7:30 给我打电话做早间简报。包含：
- 我所在城市的天气预报
- 今天的日历
- 隔夜收到的紧急邮件
- 与我兴趣相关的 3 条头条新闻

保持简洁 —— 控制在 2 分钟以内。如果我问问题，回答。
如果我没接，不要重拨。
```

## 示例：价格告警

```text
监控 NVDA 股票价格。如果一天内下跌超过 5%，
立即给我打电话并告诉我发生了什么。包含可能解释
下跌的相关新闻。
```

## 示例：紧急邮件过滤

```text
在工作时间内，每 15 分钟检查我的收件箱。
如果看到来自我老板的邮件或任何标记为紧急的邮件，
给我打电话做摘要。其他的只发聊天消息。
```

## 关键洞察

- **不要过度使用**。重点是电话通话意味着"这真的很重要"。如果智能体每天给你打 10 次电话，你就会开始忽略它。
- **双向对话**。与推送通知不同，你可以在通话中问后续问题。
- **用快速模型处理通话**。电话对话需要快速响应。如果你的 OpenClaw 支持技能级别的模型路由，将 clawr.ing 分配给快速模型。
- **隐私**。clawr.ing 不存储录音或字幕。音频在传输中加密，处理后丢弃。
