# 市场研究与产品工厂

你想做一个产品但不知道做什么。或者你有业务需要了解客户在挣扎什么。这个工作流使用 Last 30 Days 技能从 Reddit 和 X 上挖掘真实痛点，然后让 OpenClaw 为这些问题构建解决方案。

## 功能

- 使用 [Last 30 Days](https://github.com/mvanhorn/last30days-skill/) 技能研究过去 30 天 Reddit 和 X 上的任何主题
- 呈现人们发布的真实挑战、投诉和功能需求
- 帮助你从真实用户痛点中识别产品机会
- 更进一步：让 OpenClaw 构建解决某个挑战的 MVP
- 创建从研究到产品的完整流水线，无需你编码

## 痛点

大多数创业者都在"做什么"这个问题上挣扎。传统市场研究意味着数小时手动浏览论坛、社交媒体和评论网站。这个工具自动化了从发现到原型的整个流程。

## 所需技能

- [Last 30 Days](https://github.com/mvanhorn/last30days-skill/) 技能
- Telegram 或 Discord 集成用于接收研究报告

## 如何设置

1. 安装 Last 30 Days 技能：

```text
安装这个技能：https://github.com/mvanhorn/last30days-skill/
```

2. 对任何主题进行研究：

```text
请使用 Last 30 Days 技能研究人们在 [你的主题] 方面遇到的挑战。

将发现整理为：
- 排名靠前的痛点（按频率排序）
- 具体的投诉和功能需求
- 现有解决方案的差距
- 新产品的机会
```

3. 选择一个痛点并让 OpenClaw 构建解决方案：

```text
给我构建一个解决 [研究中的痛点] 的 MVP。
保持简单 —— 只要核心功能。
以 web 应用形式发布，方便分享。
```

4. 持续市场情报，安排定期研究：

```text
每周一早上，使用 Last 30 Days 技能研究人们在 [你的领域]
上在 Reddit 和 X 说什么。总结最佳机会并发送到我的 Telegram。
```

## 关键洞察

- 这是**自动驾驶的创业**：发现问题 → 验证需求 → 构建解决方案，全部通过短信。
- Last 30 Days 技能给你真实的、未经过滤的用户情感 —— 不是净化过的调查数据。
- 你不需要技术背景。OpenClaw 做研究也做构建。
