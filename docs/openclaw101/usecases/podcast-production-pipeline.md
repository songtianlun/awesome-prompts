# 播客制作流水线

你有一个播客创意，甚至有一堆待录制的主题。但在研究嘉宾、撰写大纲、起草开场白、生成节目笔记和撰写社交媒体推广帖之间 —— 制作开销杀死了你的动力。如果你只需要给一个主题，就能拿到完整的制作包呢？

这个用例将智能体串联起来，处理从主题到可发布素材的整个播客制作工作流。

## 痛点

独立播客人和小团队在制作上花的时间比实际录制还多。研究需要数小时，节目笔记常常是事后补的，社交媒体推广往往是第一个被跳过的。创造性的部分 —— 对话 —— 可能只占总工作量的 30%。这个智能体处理剩下的 70%。

## 功能

- **节目研究** —— 给定一个主题或嘉宾名字，编译背景研究、谈话要点和建议问题
- **大纲与脚本** —— 生成结构化的节目大纲，包含开场脚本、环节过渡和结束语
- **节目笔记** —— 录制后，将字幕处理成带时间戳的节目笔记，并附上提到的所有内容的链接
- **社交媒体套件** —— 为 X、LinkedIn 和 Instagram 创建推广帖子，包含节目亮点和精彩引言
- **节目描述** —— 撰写针对 Spotify、Apple Podcasts 和 YouTube 优化的 SEO 节目描述

## 所需技能

- 网页搜索/研究技能（用于嘉宾研究和主题深入）
- 文件系统访问（用于读取字幕和写入输出文件）
- Slack、Discord 或 Telegram 集成（用于交付素材）
- 可选：`sessions_spawn` 用于并行运行研究和写作智能体
- 可选：RSS 订阅技能（用于监控竞品播客）

## 如何设置

1. 录制前 —— 生成研究和大纲：

```text
我要录制一期关于 [主题] 的播客节目。我的嘉宾是 [姓名]。

请：
1. 研究嘉宾 —— 背景、近期工作、犀利观点，
   以及他们公开说过的有趣或有争议的内容。
2. 研究主题 —— 关键趋势、最新新闻、常见误解，
   以及听众可能已知和会感到惊讶的内容。
3. 生成节目大纲：
   - 冷开场钩子（1-2 句抓住注意力）
   - 开场脚本（30 秒，轻松语调）
   - 5-7 个采访问题，从简单/建立关系到深入/尖锐排序
   - 2-3 个"后备"问题，以防对话停滞
   - 结束环节，包含行动号召

保存到 ~/podcast/episodes/[节目编号]/prep/
```

2. 录制后 —— 生成节目笔记和推广内容：

```text
这是第 [编号] 期的字幕：[粘贴或指向文件]

请：
1. 撰写带时间戳的节目笔记 —— 每个主要话题转换都标注时间戳
   和一句话摘要。包含提到的所有内容的链接（工具、书籍、
   文章、人物）。
2. 撰写节目描述（最多 200 字），针对播客搜索优化。
   自然地包含 3-5 个相关关键词。
3. 创建社交媒体帖子：
   - X/Twitter：3 条推文 —— 一条精彩引言、一条关键洞察、一条
     引发讨论的问题。每条不超过 280 字符。
   - LinkedIn：1 篇帖子，专业语调，100-150 字。
   - Instagram 描述：1 篇帖子，表情符号，轻松语调，包含相关标签。
4. 提取"亮点"列表 —— 3 个最有趣/最令人惊讶的时刻
   及其时间戳。

保存到 ~/podcast/episodes/[节目编号]/publish/
```

3. 可选 —— 竞品监控：

```text
每天监控这些播客 RSS 源：
- [RSS URL 1]
- [RSS URL 2]

当有新节目涉及与我的播客相关的主题时，
通过 Telegram 发送消息：
- 节目标题和链接
- 一句话摘要
- 这是否是我应该回应或从我的角度覆盖的内容
```

## 关键洞察

- **录制前的研究**是价值最大的地方。带着深入的嘉宾研究走进采访，会让对话质量大幅提升 —— 这是后期制作无法弥补的。
- 带时间戳的节目笔记是巨大的听众留存工具。大多数播客人跳过它们因为太枯燥。这个智能体让它们毫不费力。
- 社交媒体套件节省了最多的**经常性**时间。每期节目都需要推广，结构总是相同的 —— 非常适合自动化。
- 与[多智能体内容工厂](content-factory.md)配合使用效果更好，可以将播客内容重新利用为博客文章、新闻简报或视频片段。
